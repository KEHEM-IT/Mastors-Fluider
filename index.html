üöÄ Installation

For SCSS Projects

Import the breakpoints utility in your SCSS file:

SCSS 
@use 'path/to/breakpoints' as *;

// Or with a custom namespace
@use 'path/to/breakpoints' as bp;

// Usage: @include bp.break-up(md) { ... }

For CSS Projects

Link the breakpoints utility CSS file in your HTML:

HTML 
<link rel="stylesheet" href="path/to/breakpoints.css">

Or import in your CSS file:

CSS 
@import url('path/to/breakpoints.css');
‚úÖ Installation Complete! You can now use breakpoint mixins in SCSS or utility classes in HTML.

üõ†Ô∏è SCSS Mixins & Functions

Main Breakpoint Mixin

breakpoint($size, $type, $orientation) - Full control over media queries

SCSS 
// Mobile-first (min-width) - Default
.container {
  @include breakpoint(md) { // 768px+
    padding: 2rem;
  }
}

// Desktop-first (max-width)
.header {
  @include breakpoint(lg, 'max') { // 1024px and below
    height: 60px;
  }
}

// With orientation
.gallery {
  @include breakpoint(md, 'min', landscape) {
    grid-template-columns: repeat(3, 1fr);
  }
}

Mobile-First: break-up()

Shorthand for min-width queries. Applies styles from breakpoint and up.

SCSS 
.card {
  padding: 1rem;
  
  @include break-up(md) { // 768px+
    padding: 2rem;
  }
  
  @include break-up(xl) { // 1200px+
    padding: 3rem;
  }
}

Desktop-First: break-down()

Shorthand for max-width queries. Applies styles up to breakpoint.

SCSS 
.sidebar {
  width: 300px;
  
  @include break-down(lg) { // 1024px and below
    width: 250px;
  }
  
  @include break-down(md) { // 768px and below
    width: 100%;
  }
}

Range: break-between()

Apply styles between two breakpoints.

SCSS 
.banner {
  @include break-between(md, xl) { // 768px to 1200px
    font-size: 2rem;
  }
}

Single Range: break-only()

Apply styles only within a specific breakpoint range.

SCSS 
.notification {
  @include break-only(md) { // 768px to 1023px only
    position: fixed;
  }
}

Custom Breakpoint

Use custom width values not in the predefined map.

SCSS 
.special {
  @include custom-breakpoint(50em) { // 800px+
    display: grid;
  }
  
  @include custom-breakpoint(900px, 'max') {
    display: block;
  }
}

Get Breakpoint Function

Retrieve breakpoint values for use in calculations or CSS custom properties.

SCSS 
$tablet-width: get-breakpoint(tablet); // Returns: 48em

.custom-element {
  max-width: $tablet-width;
}

// Pass to JavaScript via CSS variables
:root {
  --breakpoint-md: #{get-breakpoint(md)};
}

üé® CSS Utility Classes

Mobile-First Classes

Show elements at breakpoint and above (min-width).

Class Shows At Example
.break-up-sm 576px+ Visible on small devices and up
.break-up-md 768px+ Visible on tablets and up
.break-up-lg 1024px+ Visible on laptops and up
.hide-up-md 768px+ Hidden from tablets and up
HTML

  Visible on tablets and larger
</div>

Desktop-First Classes

Show elements at breakpoint and below (max-width).

Class Shows At Example
.break-down-md 768px and below Visible up to tablets
.break-down-lg 1024px and below Visible up to laptops
.hide-down-md 768px and below Hidden on tablets and smaller
HTML

<nav class="break-down-tablet">Mobile Navigation</nav>

<!-- Desktop navigation -->

‚úÖ Best Practices

1. Use Mobile-First Approach

‚úÖ Good: Start with mobile styles, then enhance for larger screens
SCSS 
.element {
  font-size: 1rem; // Mobile base
  
  @include break-up(md) {
    font-size: 1.5rem; // Tablet+
  }
}
‚ùå Avoid: Starting with desktop and overriding for mobile
SCSS 
.element {
  font-size: 1.5rem; // Desktop default
  
  @include break-down(md) {
    font-size: 1rem; // Override for mobile
  }
}

2. Keep Breakpoint Logic Simple

Use break-between() or break-only-* instead of nesting breakpoints.

3. Consistent Naming

Choose either core (sm, md, lg) or custom (mobile, tablet, laptop) naming. Don't mix them unnecessarily.

4. Don't Overuse Breakpoints

Not every element needs 5+ breakpoints. Focus on critical layout shifts.

5. Test Accessibility

  • Test with increased font sizes (150%, 200%)
  • Test on real devices, not just browser dev tools
  • Verify touch targets are at least 44√ó44px on mobile
  • Ensure content is accessible at all breakpoints

6. Performance Tips

  • Minimize unused CSS classes (use PurgeCSS if needed)
  • Combine related media queries when possible
  • Load critical CSS inline, defer non-critical styles
  • Use CSS containment for better rendering performance

‚ùì Frequently Asked Questions

Why em units instead of px?

Accessibility. Em units respect user browser font-size settings. If a user increases their font size for readability, breakpoints using em will adjust accordingly.

Conversion: 1em = 16px (browser default)

Can I mix SCSS and CSS classes?

Absolutely! Use SCSS mixins for custom components and CSS utility classes for rapid layout adjustments.

What's the difference between break-down and break-up?

  • break-up-md: Shows at 768px and above (mobile-first)
  • break-down-md: Shows at 768px and below (desktop-first)

Why does the CSS file use !important?

Utility classes use !important to ensure they override component styles, following the utility-first pattern. This is intentional for maximum flexibility.

Can I customize breakpoint values?

For SCSS: Yes, edit the $breakpoints map in the SCSS file.

For CSS: You'll need to manually edit the media queries or use CSS custom properties.

Which approach should I use: SCSS or CSS classes?

Use SCSS When Use CSS Classes When
Building complex components Rapid prototyping
Need fine-grained control Simple show/hide functionality
Component library development Utility-first designs
Prefer semantic CSS Quick responsive adjustments

Best Practice: Use both! SCSS for component styling, CSS classes for layout utilities.

How does this compare to Tailwind CSS?

Mastors Fluider is complementary to Tailwind. You can use:

  • Tailwind for utility-first styling (p-4, bg-blue-500)
  • Mastors Fluider for complex responsive logic and SCSS mixins

What's the file size impact?

The CSS file is approximately 15-20KB unminified. When minified and gzipped, it's only 3-4KB, which is negligible for most projects.

Can I use with JavaScript frameworks?

Yes! Works seamlessly with:

  • React (use className)
  • Vue (use :class binding)
  • Angular (use [ngClass])
  • Svelte (use class:)
  • Vanilla JavaScript (classList.add/remove)
